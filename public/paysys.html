日志存储目录配置

xxxxxxxxxx

微信支付API

Native支付API

  • 请求URL: http://主机地址:自定义端口号/wechat/native

  • 请求方式:POST

  • 请求参数(JSON格式)

参数名 变量 必填 类型[长度限制] 描述
应用ID appid 否(未填使用系统默认配置的appid) string[1,32] 由微信生成的应用ID，全局唯一。请求基础下单接口时请注意APPID的应用属性，例如公众号场景下，需使用应用属性为公众号的APPID
直连商户号 mchid 否(未填使用系统默认配置的mchid) string[1,32] 直连商户的商户号，由微信支付生成并下发。
商品描述 description 是 string[1,127] 商品描述
示例值：Image形象店-深圳腾大-QQ公仔
商户订单号 out_trade_no 是 string[6,32] 商户系统内部订单号，只能是数字、大小写字母_-*且在同一个商户号下唯一
交易结束时间 time_expire 否 string[1,64] 订单失效时间，遵循rfc3339标准格式，格式为YYYY-MM-DDTHH:mm:ss+TIMEZONE，YYYY-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日 13点29分35秒。
订单失效时间是针对订单号而言的，由于在请求支付的时候有一个必传参数prepay_id只有两小时的有效期，所以在重入时间超过2小时的时候需要重新请求下单接口获取新的prepay_id。其他详见时间规则。
建议：最短失效时间间隔大于1分钟
示例值：2018-06-08T10:34:56+08:00
通知地址 notify_url 否(未填使用系统默认) string[1,256] 通知URL必须为直接可访问的URL，不允许携带查询串，要求必须为https地址。
格式：URL
示例值：https://www.后端接收支付结果的地址
订单优惠标记 goods_tag 否 string[1,32] 订单优惠标记
示例值：WXG
订单金额 amount 是 object 参数名 变量 必填 类型[长度限制] 描述
总金额 total 是 int 订单总金额，单位为分。
货币类型 currency 否 string[1,16] CNY：人民币，境内商户号仅支持人民币。
优惠功能 detail 否 object 参数名 变量 必填 类型[长度限制] 描述
订单原价 cost_price 否 int 1、商户侧一张小票订单可能被分多次支付，订单原价用于记录整张小票的交易金额。
2、当订单原价与支付金额不相等，则不享受优惠。
3、该字段主要用于防止同一张小票分多次支付，以享受多次优惠的情况，正常支付订单不必上传此参数。
商品小票ID invoice_id 否 string[1,32] 商家小票ID
单品列表 goods_detail 否 array 参数名 变量 必填 类型[长度限制] 描述
商户侧商品编码 merchant_goods_id 是 string[1,32] 由半角的大小写字母、数字、中划线、下划线中的一种或几种组成。
微信支付商品编码 wechatpay_goods_id 否 string[1,32] 微信支付定义的统一商品编号（没有可不传）
商品名称 goods_name 否 string[1,256] 商品的实际名称
商品数量 quantity 是 int 用户购买的数量
商品单价 unit_price 是 int 商品单价，单位为分
场景信息 scene_info 否 object 参数名 变量 必填 类型[长度限制] 描述
用户终端IP payer_client_ip 是 string[1,45] 用户的客户端IP，支持IPv4和IPv6两种格式的IP地址。
商户端设备号 device_id 否 string[1,32] 商户端设备号（门店号或收银设备ID）
商户门店信息 store_info 否 object 参数名 变量 必填 类型[长度限制] 描述
门店编号 id 是 string[1,32] 商户侧门店编号
门店名称 name 否 string[1,256] 商户侧门店名称
地区编码 area_code 否 string[1,32] 地区编码，详细请见省市区编号对照表。
详细地址 address 否 string[1,512] 详细的商户门店地址
结算信息 settle_info 否 object 参数名 变量 必填 类型[长度限制] 描述
是否指定分账 profit_sharing 否 boolean 是否指定分账

申请退款API

  • 请求URL: http://主机地址:自定义端口号/wechat/refund
  • 请求方式:POST
  • 请求参数
参数名称 变量 必填 类型[长度] 描述
微信支付订单号 transaction_id 二选一 string[1, 32] 原支付交易对应的微信订单号
商户订单号 out_trade_no string[6, 32] 原支付交易对应的商户订单号
商户退款单号 out_refund_no 是 string[1, 64] 商户系统内部的退款单号，商户系统内部唯一，只能是数字、大小写字母_-|*@ ，同一退款单号多次请求只退一笔。
退款原因 reason 否 string[1, 80] 若商户传入，会在下发给用户的退款消息中体现退款原因
退款结果回调url notify_url 否 string[8, 256] 异步接收微信支付退款结果通知的回调地址，通知url必须为外网可访问的url，不能携带参数。 如果参数中传了notify_url，则商户平台上配置的回调地址将不会生效，优先回调当前传的这个地址。
退款资金来源 funds_account 否 string[1,32] 若传递此参数则使用对应的资金账户退款，否则默认使用未结算资金退款（仅对老资金流商户适用）
枚举值：
AVAILABLE：可用余额账户
金额信息 amount 是 object 参数名 变量 必填 类型[长度限制] 描述
退款金额 refund 是 int 退款金额，单位为分，只能为整数，不能超过原订单支付金额。
退款出资账户及金额 from 否 array 参数名 变量 必填 类型[长度限制] 描述
出资账户类型 account 是 string[1, 32] 下面枚举值多选一。
枚举值：
AVAILABLE : 可用余额
UNAVAILABLE : 不可用余额
出资金额 amount 是 int 对应账户出资金额
原订单金额 total 是 int 原支付交易的订单总金额，单位为分，只能为整数。
退款币种 currency 是 string[1, 16] 符合ISO 4217标准的三位字母代码，目前只支持人民币：CNY。
退款商品 goods_detail 否 array 参数名 变量 必填 类型[长度限制] 描述
商户侧商品编码 merchant_goods_id 是 string[1, 32] 由半角的大小写字母、数字、中划线、下划线中的一种或几种组成
微信支付商品编码 wechatpay_goods_id 否 string[1, 32] 微信支付定义的统一商品编号（没有可不传）
商品名称 goods_name 否 string[1, 256] 商品的实际名称
商品单价 unit_price 是 int 商品单价金额，单位为分
商品退款金额 refund_amount 是 int 商品退款金额，单位为分
商品退货数量 refund_quantity 是 int 单品的退款数量
