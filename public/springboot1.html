静态资源访问

静态资源目录:

只要静态资源放在 类路径(resources文件夹)下 /static (or /public or /resources or /META-INF/resources)

我们就可以通过 当前项目根路径/ + 静态资源名称 来访问静态资源

原理: 静态资源映射/**

请求进来先去找controller 看能不能处理,不能处理的所有请求又都交给静态资源处理器,静态资源就回去上面那几个目录中找.要是静态资源处理器也没找到,就会报404

  • 静态资源访问前缀(默认没有前缀的,但是一般配置一个前缀)

    • 配置方法:spring.mvc.static-path-pattern=/resources/**( properties)

      • xxxxxxxxxx
        spring:
          mvc:
            static-path-pattern: /res/**

        以上配置代表所有的以res开头的请求都代表静态资源请求,这样spring 看到就不会先找controller 而是直接交给静态资源处理器

  • 静态资源的默认路径也可以改

    • 使用spring.web.resources.static-locations=classpath:/abc/ 或 [classpath:/abc/,....] 多个写法 properties
    • 把之前的默认的静态文件路径改为类路径下的abc文件夹中
    • 注意: 有时候书写正确,但是就是找不到,需要改一改静态文件名

接收前端POST请求发送的raw JSON 和 form-data的方法(重点)

  • @RequestBody : 获取请求体(请求体格式必须是ROW)(post请求才有请求体).

  • 注意: @RequestBody 的方式请求的,每一个请求必须是通过实体对象(row:JSON)进行传参，不能通过form-data 表单提交的方式进行传参；

  • 此时想要获取formdata 中的数据,得使用@RequestParam("key")

  • 总结

  • 前端POST传送的是raw : JSON 格式的数据的话 使用一个属性与JSON中key一一对应的类对象接收(spring 会自动转换),要使用@RequestBody注解

  • 前端POST传送的是form-data:

    • 1.可以使用一个与form-data中key一一对应的类对象接收(spring 会自动转换),但是不能写注解

    • 2.可以使用@RequestParam("keyValue") 注解 像获取get请求参数一样来获取form-data中的每一项

路径中传递参数的方式:矩阵变量(高级感)

  • 以前方式:/cars/{path}?param1=xxx&param2=zzz&param3=ccc 使用@RequestPath 和 @RequestParam 分别获取path和 参数1,2,3

  • 现在新方式 : /cars/{path};param1=a;param2=c;param3=aaa,bbb,ccc; 这种方式称为矩阵变量

  • 矩阵变量必须有url路径变量才能被解析,因为参数和路径变量绑定

  • 启用矩阵变量方式需要设置如下代码

    • xxxxxxxxxx
      @Configuration(proxyBeanMethods = false)
      public class Config {
          @Bean
          public WebMvcConfigurer webMvcConfigurer(){
              return new WebMvcConfigurer(){
                  @Override
                  public void configurePathMatch(PathMatchConfigurer configurer){
                      UrlPathHelper urlPathHelper = new UrlPathHelper();
                      urlPathHelper.setRemoveSemicolonContent(false);
                      configurer.setUrlPathHelper(urlPathHelper);
                  }
              };
          }
      }
